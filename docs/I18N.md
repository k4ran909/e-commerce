## Usage Guide

### 1. Basic Translation

```tsx
import { useTranslation } from 'react-i18next';

function MyComponent() {
  const { t } = useTranslation();
  
  return (
    <div>
      <h1>{t('common.welcome')}</h1>
      <button>{t('common.submit')}</button>
    </div>
  );
}
```

### 2. Nested Translations

```tsx
// Access nested keys with dot notation
<h2>{t('home.hero.title')}</h2>
<p>{t('products.noProducts')}</p>
<span>{t('auth.login.email')}</span>
```

### 3. Interpolation (Dynamic Values)

**Translation file:**
```json
{
  "validation": {
    "minLength": "Must be at least {{min}} characters"
  }
}
```

**Component:**
```tsx
<p>{t('validation.minLength', { min: 8 })}</p>
// Output: "Must be at least 8 characters"
```

### 4. Pluralization

**Translation file:**
```json
{
  "items": "{{count}} item",
  "items_plural": "{{count}} items"
}
```

**Component:**
```tsx
<p>{t('items', { count: 1 })}</p>  // "1 item"
<p>{t('items', { count: 5 })}</p>  // "5 items"
```

### 5. Language Switching

**Using Language Selector:**
```tsx
import { LanguageSelector } from '@/components/language-selector';

function Header() {
  return <LanguageSelector />;
}
```

**Programmatic Change:**
```tsx
const { i18n } = useTranslation();

// Switch to Spanish
i18n.changeLanguage('es');

// Get current language
const currentLang = i18n.language; // 'en', 'es', 'fr', or 'id'
```

### 6. Conditional Text

```tsx
const { t } = useTranslation();

const statusText = order.isDelivered 
  ? t('orderStatus.delivered') 
  : t('orderStatus.pending');
```

### 7. Array Mapping

```tsx
const { t } = useTranslation();

const categories = ['rings', 'necklaces', 'earrings', 'bracelets'];

return (
  <ul>
    {categories.map(cat => (
      <li key={cat}>{t(`products.${cat}`)}</li>
    ))}
  </ul>
);
```

```tsx

```

‚úÖ **Good:**
```tsx
<button>{t('common.submit')}</button>
```

### 3. Consistent Key Naming

‚úÖ **Good Examples:**
```json
{
  "common.save": "Save",
  "common.cancel": "Cancel",
  "auth.login.title": "Login",
  "products.addToCart": "Add to Cart"
}
```

‚ùå **Bad Examples:**
```json
{
  "btn1": "Save",                    // Not descriptive
  "loginPageHeader": "Login",        // Use dot notation
  "add_to_cart": "Add to Cart",      // Use camelCase
  "msg": "Required"                  // Too abbreviated
}
```

### 4. Use Interpolation for Dynamic Content

```tsx
// Instead of concatenation
const text = t('greeting') + ' ' + userName;

// Use interpolation
const text = t('greeting.withName', { name: userName });
```

### 5. Keep Translations DRY

Reuse common translations:
```json
{
  "common": {
    "save": "Save",
    "cancel": "Cancel"
  }
}
```

```tsx
// Reuse across components
<button>{t('common.save')}</button>
<button>{t('common.cancel')}</button>
```

## Troubleshooting

### Issue: Translation key not found

**Symptoms:** Display shows the key instead of translated text (e.g., "auth.login.title")

**Solutions:**
1. Check if the key exists in the translation file
2. Verify the key path is correct (case-sensitive)
3. Ensure the translation file is imported in `i18n.ts`
4. Check for typos in the key name

### Issue: Translation works in English but not other languages

**Solutions:**
- Check if the key exists in all language files
- Verify JSON syntax is correct (no trailing commas)
- Ensure all language files are properly imported

### Issue: Language doesn't change

**Solutions:**
1. Check if `i18n.changeLanguage()` is being called correctly
2. Verify the language code is supported
3. Clear browser cache and localStorage
4. Check browser console for errors

### Issue: Language not persisting

**Solutions:**
- Check browser's localStorage permissions
- Verify `caches: ['localStorage']` in i18n config
- Check if cookies/storage are being blocked
